# Rumia plan archive

These documents are retained for historical context and git provenance. They
are not execution authority.

## Current index

The active, supporting, deferred, and evidence documents are tracked in
[`../PLAN-INDEX.md`](../PLAN-INDEX.md). That index is the only authority map;
this file describes the archive itself.

The active execution set is intentionally small:

- the Portugal-wide activity-first master plan;
- the single 2026-07-14 frontend polish plan, whose visual acceptance is open;
- supporting domain, runtime, and optional-map contracts.

The VPS migration and 2026-07-12 deep-redesign implementation plans are
archived after their bounded private-release work. The map Phase 2/3 packet is
still deferred behind owner/legal/provider decisions. None is an invitation to
execute old unchecked tasks.

The active supporting specs are the activity-curation, activity-map,
VPS-platform, and frontend-aesthetic design specs under
`docs/superpowers/specs/`.

## Archived plan families

- `2026-07-09-choice-led-*`: initial choice-led implementation/spec.
- `2026-07-10-rumia-full-rework-*`: route-first product master and phase plans.
- `2026-07-11-rumia-full-redesign-and-ui-plan.md`: superseded UI correctness
  baseline, consolidated into the deep redesign plan.
- `2026-07-11-rumia-vps-self-hosted-migration.md`: completed private VPS
  migration and cutover history.
- `2026-07-12-rumia-frontend-*`: overlapping aesthetic, implementation, and
  foundation slices, plus the completed deep-redesign implementation history.

The archive is reversible. Restore a document only as a historical reference;
do not execute its unchecked tasks without first reconciling them into an
active authority, updating [`../PLAN-INDEX.md`](../PLAN-INDEX.md), and recording
fresh evidence.
